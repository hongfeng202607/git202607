三、详细安装步骤

3.1 安装 Node.js

1
访问 nodejs.org，下载 LTS 版本（≥ 20.x）
2
安装时勾选"Add to PATH"，一路 Next 完成
3
打开命令行验证：
node -v    # 应显示 v20.x.x 或更高
npm -v     # 应显示 9.x.x 或更高

3.2 安装 PostgreSQL

1
下载 PostgreSQL 18 安装包（官方下载）
2
安装时注意：
- 安装路径：F:\PostgreSQL\18（本例）
- 端口：5432（默认）
- 超级用户密码：设置密码（本例为 123456）
- Locale：选择"Chinese, Simplified China" 或默认
3
验证安装：
"F:\PostgreSQL\18\bin\psql.exe" -U postgres -V

3.3 安装项目依赖

1
进入项目目录：
cd E:\Desktop\工作汇报系统
2
安装后端依赖：
cd backend
npm install
3
安装前端依赖：
cd ../frontend
npm install

3.4 初始化数据库

确保 PostgreSQL 服务已启动（可通过 Windows 服务管理器或重启电脑启动）。
1
创建数据库并执行初始化脚本：
"F:\PostgreSQL\18\bin\psql.exe" -U postgres -c "CREATE DATABASE work_report_db;"
"F:\PostgreSQL\18\bin\psql.exe" -U postgres -d work_report_db -f "E:\Desktop\工作汇报系统\database\init.sql"
2
验证表是否创建成功：
"F:\PostgreSQL\18\bin\psql.exe" -U postgres -d work_report_db -c "\dt"
应显示 9 张表：ai_config, audit_log, cycle_report, department, notification, report_record, report_relation, sys_user, work_record
如果使用 Node.js 执行 init.sql 而非 psql，确保文件编码为 UTF-8。

3.5 配置数据库连接

编辑 backend/config/index.js，确认数据库连接参数：

db: {
  host: process.env.DB_HOST || '127.0.0.1',
  port: parseInt(process.env.DB_PORT || '5432'),
  database: process.env.DB_NAME || 'work_report_db',
  user: process.env.DB_USER || 'postgres',
  password: process.env.DB_PASSWORD || '123456',  // ← 改为你设置的密码
}

也可以通过环境变量覆盖（推荐）：

set DB_PASSWORD=你的密码
set JWT_SECRET=你的密钥

3.6 启动服务

开发模式（推荐）：

1
直接双击 start.bat，或命令行执行：
cd E:\Desktop\工作汇报系统
start.bat

生产模式：

1

cd E:\Desktop\工作汇报系统\frontend

浏览器访问 http://localhost:8902，应看到登录页面。

五、运维管理

5.1 启动与停止

操作方式
启动服务双击 start.bat 或 start-prod.bat
停止服务双击 stop.bat（强制杀死所有 node 进程）
检查运行状态访问 http://localhost:8902/api/health

5.2 防火墙配置（Windows）

其他电脑访问需要放行 8902 端口：

控制面板 → Windows Defender 防火墙 → 高级设置
→ 入站规则 → 新建规则 → 端口 → TCP → 8902 → 允许连接

5.3 日志查看

后端日志输出在控制台窗口。可通过 morgan 查看 HTTP 请求日志，格式为 :method :url :status :response-time ms。

5.4 常见问题

Q：启动报"端口被占用"？
A：双击 stop.bat 杀死所有 node 进程后重试。或用 netstat -ano | findstr :8902 查看占用进程的 PID，手动结束。
Q：数据库连接失败？
A：确认 PostgreSQL 服务已启动（任务管理器 → 服务 → postgresql-x64-18），检查连接密码是否匹配。
Q：前端页面空白/报错？
A：运行 cd frontend && npx vite build 重新构建前端，然后重启后端。
Q：其他同事无法访问？
A：确保防火墙放行了 8902 端口；确认本机局域网 IP（ipconfig）；同事访问 http://你的IP:8902。

六、AI 功能配置

6.1 配置方法

使用 admin 账号登录 → 侧边栏"AI 配置" → 填写：

字段说明示例
API 地址AI 服务的完整 API URLhttp://127.0.0.1:8080/v1/chat/completions
模型名称使用的模型Qwen3-1.7B
API 密钥认证 Token（按需）sk-lm-xxx:yyy
系统提示词AI 行为指令"你是一个工作汇报助手..."

6.2 支持模型

  • OpenAI 兼容 API（包括智谱 GLM、DeepSeek、通义千问、Kimi 等国内模型）
  • 本地模型（LM Studio、Ollama 等本地推理服务）
  • Qwen3 系列（自动适配思考模式）

6.3 注意事项

Node.js 的 fetch 对 localhost 可能解析到 IPv6，建议使用 127.0.0.1 而非 localhost。
