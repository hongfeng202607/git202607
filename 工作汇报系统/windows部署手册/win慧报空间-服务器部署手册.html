端口用途对内/对外

安装 Node.js 22 LTS

以下以 Ubuntu 22.04 为例。CentOS / Windows Server 命令见附录。

# 更新包管理器 sudo apt update && sudo apt upgrade -y # 安装 curl（如果没有） sudo apt install -y curl # 添加 NodeSource 仓库并安装 Node.js 22 LTS curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - sudo apt install -y nodejs # 验证版本 node -v # 应显示 v22.x npm -v # 应显示 10.x
📌 验证方法 如果 node -v 显示版本号，则安装成功。

安装 PM2（进程管理工具）

sudo npm install -g pm2 pm2 -v # 应显示 5.x
3

安装 PostgreSQL & 创建数据库

3.1 安装 PostgreSQL 16

# 安装 PostgreSQL 16 sudo apt install -y postgresql-16 # 确认安装成功 psql --version # 应显示 psql (PostgreSQL) 16.x # 查看服务状态 sudo systemctl status postgresql # 应显示 active (running)
⚠️ 如果 postgresql-16 找不到 Ubuntu 22.04 默认源可能只有 14 版本。
改用 sudo apt install postgresql（会安装默认版本）即可。

3.2 设置数据库密码

# 切换到 postgres 系统用户 sudo -i -u postgres # 进入 psql 命令行 psql # 设置 postgres 用户密码（替换 YourStrongPassword! 为你的密码） ALTER USER postgres WITH PASSWORD 'YourStrongPassword!'; # 退出 psql \q # 退出 postgres 用户 exit
🔐 重要 生产环境绝对不能使用默认密码 123456！ 建议用 openssl rand -base64 16 生成随机密码。

3.3 创建数据库和专用用户

建议不要直接用 postgres 超级用户连接应用，而是创建一个专用用户：

sudo -i -u postgres psql -- 创建数据库 CREATE DATABASE work_report_db; -- 创建专用用户（密码替换为你的强密码） CREATE USER huibao_user WITH PASSWORD 'HuibaoPass123!'; -- 授予权限 GRANT ALL PRIVILEGES ON DATABASE work_report_db TO huibao_user; -- 退出 \q exit
📌 后续配置时 在环境变量中设置：DB_USER=huibao_user 和 DB_PASSWORD=HuibaoPass123!

3.4 配置 PostgreSQL 仅监听本地

# 编辑配置文件 sudo vim /etc/postgresql/16/main/postgresql.conf # 找到并修改 listen_addresses listen_addresses = 'localhost' # 只监听本地，不暴露到公网 # 重启生效 sudo systemctl restart postgresql
⚠️ 安全红线 如果 listen_addresses = '*'，数据库会暴露在公网，极其危险！ 一定要改成 'localhost' 或 '127.0.0.1'。

上传代码 & 安装依赖

4.1 将代码传到服务器

方式一：从本机拷贝（推荐第一次部署用）

# 在本地打包（Windows 上用 Git Bash 或 PowerShell） cd E:\Desktop\工作汇报系统 tar -czf huibao.tar.gz --exclude=node_modules --exclude=frontend/node_modules --exclude=frontend/dist --exclude=backend/backup . # 上传到服务器（替换为你的服务器 IP） scp huibao.tar.gz root@你的服务器IP:/opt/ # 在服务器上解压 ssh root@你的服务器IP cd /opt tar -xzf huibao.tar.gz -C /opt/huibao

方式二：从 Git 仓库拉取（推荐后续更新用）

# 先在本地把代码推送到 Git 仓库（ GitHub / Gitee / 自建 Git ） # 在服务器上拉取 cd /opt git clone https://github.com/你的仓库/huibao-space.git huibao # 如果用 SSH 方式：git clone git@github.com:你的仓库/huibao-space.git huibao

4.2 安装后端依赖

cd /opt/huibao/backend npm install --production # 安装完成后，node_modules 目录大约 8MB

4.3 安装前端依赖（仅构建时需要）

cd /opt/huibao/frontend npm install # 前端依赖较大，约 200MB，但构建后可以删除 node_modules

初始化数据库表结构

5.2 验证表结构

# 连接到数据库查看 sudo -u postgres psql -d work_report_db # 列出所有表 \dt # 应看到 9 张表： # ai_config, audit_log, cycle_report, department, # notification, report_record, report_relation, # sys_user, work_record # 查看表结构示例 \d sys_user # 退出 \q

如果你有备份数据或演示数据需要导入：

配置环境变量 & JWT 密钥

6.1 生成 JWT 密钥

# 生成一个 64 字符的随机密钥 openssl rand -hex 32 | tee /opt/huibao/backend/jwt-secret.txt # 输出类似：a1b2c3d4e5f6...（共 64 个十六进制字符）

6.2 创建环境变量脚本

创建 /opt/huibao/backend/.env.sh，写入以下内容：

#!/bin/bash # 慧报空间 - 环境变量配置 # 在启动前 source 此文件：source .env.sh export PORT=8902 # --- 数据库配置（按实际修改） --- export DB_HOST=127.0.0.1 export DB_PORT=5432 export DB_NAME=work_report_db export DB_USER=huibao_user export DB_PASSWORD=你的强密码 # --- JWT (如果设置了环境变量，会优先使用，否则读 jwt-secret.txt) --- # export JWT_SECRET=你的64位随机字符串
# 设置权限 chmod 600 /opt/huibao/backend/.env.sh

6.3 一次性设置（推荐直接用 systemd 或 PM2 的 env 配置）

实际上最简单的方式是用 PM2 的 ecosystem 文件来管理环境变量，见步骤 9。

8.2 验证服务

新开一个终端连接服务器，执行：

# 1. 健康检查 curl http://127.0.0.1:8902/api/health # 应返回：{"code":200,"msg":"OK","time":"..."} # 2. 前端页面 curl http://127.0.0.1:8902 # 应返回 HTML 内容（慧报空间的登录页） # 3. API 登录测试 curl -s http://127.0.0.1:8902/api/auth/login \ -H "Content-Type: application/json" \ -d '{"username":"admin","password":"admin123"}' # 应返回包含 token 的 JSON
⚠️ 注意 现在后端是直接 node app.js 运行的，关闭终端后服务就停了。 必须用 PM2 做守护进程（下一步）。

PM2 守护进程

9.1 创建 PM2 配置文件

创建 /opt/huibao/backend/ecosystem.config.js：

module.exports = { apps: [{ name: 'huibao', script: 'app.js', cwd: '/opt/huibao/backend', // 环境变量（修改为你的实际值） env: { NODE_ENV: 'production', PORT: 8902, DB_HOST: '127.0.0.1', DB_PORT: 5432, DB_NAME: 'work_report_db', DB_USER: 'huibao_user', DB_PASSWORD: '你的强密码', }, // 日志 error_file: '/var/log/huibao/error.log', out_file: '/var/log/huibao/out.log', log_date_format: 'YYYY-MM-DD HH:mm:ss', // 自动重启 max_restarts: 10, restart_delay: 3000, // 内存限制（超过自动重启） max_memory_restart: '500M', }] };
# 创建日志目录 sudo mkdir -p /var/log/huibao sudo chown -R $(whoami) /var/log/huibao

9.2 启动 & 设置开机自启

# 启动 cd /opt/huibao/backend pm2 start ecosystem.config.js # 保存进程列表（重启服务器后恢复） pm2 save # 设置开机自启 pm2 startup # 会输出一行 sudo 命令，复制执行即可

9.3 PM2 常用命令

命令说明
pm2 status查看所有进程状态
pm2 logs huibao查看实时日志（Ctrl+C 退出）
pm2 restart huibao重启
pm2 stop huibao停止
pm2 delete huibao删除进程
pm2 monit监控面板（CPU/内存/请求）
10

Nginx 反向代理 & HTTPS

10.1 安装 Nginx

sudo apt install -y nginx sudo systemctl start nginx sudo systemctl enable nginx

10.2 配置反向代理

创建 /etc/nginx/sites-available/huibao：

server { # 监听 80 端口（HTTP） listen 80; server_name huibao.yourcompany.com; # 改为你的域名 # 后端 API 转发 location /api/ { proxy_pass http://127.0.0.1:8902; proxy_set_header Host \$host; proxy_set_header X-Real-IP \$remote_addr; proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto \$scheme; # 超时设置 proxy_connect_timeout 30s; proxy_read_timeout 60s; proxy_send_timeout 60s; } # 前端静态文件和 SPA 路由 location / { proxy_pass http://127.0.0.1:8902; proxy_set_header Host \$host; proxy_set_header X-Real-IP \$remote_addr; proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto \$scheme; } }
# 启用站点 sudo ln -s /etc/nginx/sites-available/huibao /etc/nginx/sites-enabled/ # 测试配置是否正确 sudo nginx -t # 重新加载 Nginx sudo systemctl reload nginx

10.3 配置 HTTPS（使用 Let's Encrypt 免费证书）

# 安装 Certbot sudo apt install -y certbot python3-certbot-nginx # 自动获取证书并配置 HTTPS sudo certbot --nginx -d huibao.yourcompany.com # 按照提示输入邮箱、同意协议即可 # Certbot 会自动修改 Nginx 配置，添加 SSL 相关指令 # 并设置定时续期（证书有效期90天，自动续期） # 测试自动续期是否正常工作 sudo certbot renew --dry-run

10.4 最终 Nginx 配置效果

Certbot 配置完成后，你的 Nginx 配置会自动变成类似这样：

server { listen 443 ssl; server_name huibao.yourcompany.com; ssl_certificate /etc/letsencrypt/live/huibao.yourcompany.com/fullchain.pem; ssl_certificate_key /etc/letsencrypt/live/huibao.yourcompany.com/privkey.pem; # ... SSL 配置 ... location /api/ { proxy_pass http://127.0.0.1:8902; ... } location / { proxy_pass http://127.0.0.1:8902; ... } } server { listen 80; server_name huibao.yourcompany.com; return 301 https://\$host\$request_uri; # 自动跳转 HTTPS }

安全加固清单

⚠️ 以下每一项都必须做，缺一不可
#检查项操作状态
1 数据库密码 已改为强密码，非默认 123456 ☐
2 JWT 密钥 删掉默认 jwt-secret.txt，用环境变量 JWT_SECRET ☐
3 数据库监听地址 listen_addresses = 'localhost'，不暴露公网 ☐
4 防火墙 只开放 22(SSH)、80(HTTP)、443(HTTPS) ☐
5 HTTPS 通过 Certbot 配置 Let's Encrypt 证书 ☐
6 创建专用数据库用户 不使用 postgres 超级用户连接应用 ☐
7 定时备份数据库 配置 crontab 每天凌晨自动备份 ☐
8 系统更新 sudo apt update && sudo apt upgrade -y ☐
9 SSH 密钥登录 禁用密码登录，仅允许密钥登录 ☐

防火墙配置（UFW）

# 安装 UFW sudo apt install -y ufw # 设置默认规则 sudo ufw default deny incoming sudo ufw default allow outgoing # 开放必要端口 sudo ufw allow 22/tcp # SSH sudo ufw allow 80/tcp # HTTP sudo ufw allow 443/tcp # HTTPS # 启用防火墙 sudo ufw enable # 查看状态 sudo ufw status

定时备份（crontab）

# 编辑 crontab crontab -e # 添加以下行（每天凌晨3点备份） 0 3 * * * PGPASSWORD='你的密码' pg_dump -h 127.0.0.1 -U huibao_user -d work_report_db --no-owner > /backup/huibao_$(date +\%Y\%m\%d).sql 2>> /backup/backup.log # 保留最近30天的备份，删除更早的 0 4 * * * find /backup -name "huibao_*.sql" -mtime +30 -delete

附录：一键部署脚本

以下脚本包含了上面所有步骤，在全新 Ubuntu 22.04 服务器上直接执行即可完成部署：

#!/bin/bash # 慧报空间 - 一键部署脚本（Ubuntu 22.04） # 用法：chmod +x deploy.sh && sudo ./deploy.sh # 注意：执行前先将代码上传到 /opt/huibao set -e DB_PASSWORD="YourStrongPassword123!" JWT_SECRET=$(openssl rand -hex 32) DOMAIN="huibao.yourcompany.com" # 改为你的域名 echo "=== 1. 安装 Node.js 22 ===" curl -fsSL https://deb.nodesource.com/setup_22.x | bash - apt install -y nodejs npm install -g pm2 echo "=== 2. 安装 PostgreSQL ===" apt install -y postgresql systemctl start postgresql systemctl enable postgresql echo "=== 3. 创建数据库和用户 ===" sudo -u postgres psql <<EOF CREATE DATABASE work_report_db; CREATE USER huibao_user WITH PASSWORD '${DB_PASSWORD}'; GRANT ALL PRIVILEGES ON DATABASE work_report_db TO huibao_user; \c work_report_db GRANT ALL ON ALL TABLES IN SCHEMA public TO huibao_user; GRANT ALL ON ALL SEQUENCES IN SCHEMA public TO huibao_user; EOF echo "=== 4. 配置仅监听本地 ===" sed -i "s/^listen_addresses = '.*'/listen_addresses = 'localhost'/" /etc/postgresql/*/main/postgresql.conf systemctl restart postgresql echo "=== 5. 导入表结构 ===" PGPASSWORD=${DB_PASSWORD} psql -h 127.0.0.1 -U huibao_user -d work_report_db -f /opt/huibao/database/init.sql echo "=== 6. 安装后端依赖 ===" cd /opt/huibao/backend npm install --production echo "=== 7. 配置 JWT 密钥 ===" echo -n "$JWT_SECRET" > /opt/huibao/backend/jwt-secret.txt echo "=== 8. 构建前端 ===" cd /opt/huibao/frontend npm install npx vite build rm -rf node_modules echo "=== 9. 启动服务 ===" cd /opt/huibao/backend cat > ecosystem.config.js <<EOF module.exports = { apps: [{ name: 'huibao', script: 'app.js', env: { PORT: 8902, DB_HOST: '127.0.0.1', DB_PORT: 5432, DB_NAME: 'work_report_db', DB_USER: 'huibao_user', DB_PASSWORD: '${DB_PASSWORD}', }, max_memory_restart: '500M', }] }; EOF pm2 start ecosystem.config.js pm2 save pm2 startup echo "=== 10. 安装 Nginx ===" apt install -y nginx cat > /etc/nginx/sites-available/huibao <<'EOF' server { listen 80; server_name _; location /api/ { proxy_pass http://127.0.0.1:8902; proxy_set_header Host \$host; proxy_set_header X-Real-IP \$remote_addr; } location / { proxy_pass http://127.0.0.1:8902; proxy_set_header Host \$host; proxy_set_header X-Real-IP \$remote_addr; } } EOF ln -sf /etc/nginx/sites-available/huibao /etc/nginx/sites-enabled/ rm -f /etc/nginx/sites-enabled/default nginx -t && systemctl reload nginx echo "" echo "============================================" echo " 部署完成！" echo " 访问：http://服务器IP" echo " 登录：admin / admin123" echo " 数据库密码：${DB_PASSWORD}" echo " JWT密钥已保存到 jwt-secret.txt" echo "============================================" echo "" echo " 后续操作：" echo " 1. 配置域名：修改 /etc/nginx/sites-available/huibao 中的 server_name" echo " 2. 配置 HTTPS：sudo certbot --nginx" echo " 3. 配置防火墙：sudo ufw allow 22,80,443/tcp && sudo ufw enable"
