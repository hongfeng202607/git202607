📖 系统说明

慧报空间是面向团队的工作记录与周期汇报管理平台，支持工作记录管理、周期汇报提交与审批、AI 智能生成汇报内容，实现汇报流程的数字化闭环。


项目说明
系统名称慧报空间
前端地址http://服务器IP:5173
后端端口8902
默认管理员用户名 admin，密码 admin123
技术栈Vue 3 + Element Plus（前端）/ Express 5 + PostgreSQL 18（后端）
核心能力工作记录 · 周期汇报 · AI 生成 · 审批流转 · 部门管理 · 数据备份

🔑 登录与账号

登录

  1. 打开浏览器
    访问系统前端地址（如 http://服务器IP:5173），进入登录页
  2. 输入账号密码
    填写用户名和密码，可勾选「记住密码」方便下次登录
  3. 进入系统
    登录成功自动跳转至工作台。如果已登录则直接进入系统，不会停留在登录页

修改密码

点击右上角头像 → 选择「修改密码」，弹出弹窗。需输入原密码、新密码和确认密码。首次登录建议立即修改初始密码。忘记密码请联系超级管理员重置。

退出登录

点击右上角头像 → 选择「退出登录」。系统使用 JWT Token 认证，Token 过期后需重新登录。

💡 右上角头像菜单显示当前用户名和角色名（普通员工 / 管理者 / 超级管理员），方便确认登录身份。

✏️ 工作记录

工作记录是系统的基础数据单元，记录每日工作内容。提交后自动归入对应周期的汇报草稿。

新增记录

  1. 点击「新增记录」
    在工作记录页面右上角点击按钮，弹出新增表单
  2. 填写信息
    选择工作日期（默认当天）并填写工作内容
  3. 选择保存方式
    保存草稿 保存为草稿，稍后可编辑或提交
    保存并提交 直接提交，自动归入对应周期的汇报

记录状态与操作

状态可执行操作说明
草稿 编辑、提交、删除 未提交，可自由修改
已提交 撤回 已提交到服务器；撤回后恢复为草稿

如果工作记录已被周期汇报引用且该汇报处于待审核/已通过状态（即已锁定），则编辑/提交/删除按钮会禁用并提示原因。

列表与筛选

功能说明
日期范围筛选自定义起止日期，过滤范围内的记录
状态筛选按「草稿」或「已提交」过滤
分页每页 10/20/50/100 条，有首页/上页/下页按钮，支持点击页码跳转
导出 Excel将当前筛选结果导出为 .xlsx 文件到本地
导出 JSON将当前筛选结果导出为 JSON 备份文件
导入记录选择 JSON 文件批量导入工作记录到当前账号

记录编号

每条工作记录有唯一编号（display_id），格式为 YYMMDDHHmm（如 2606011015 表示 2026-06-01 10:15 创建的记录），在表格中显示。

📌 提交工作记录时，系统自动计算记录所在自然周（周一至周日），将本周所有已提交记录归入对应周报的草稿中。

📋 周期汇报

周期汇报是向上级提交的定期工作总结，支持周报和月报两种类型。

新建汇报

  1. 点击「新建汇报」
    在周期汇报页面点击右上角的按钮
  2. 选择类型和周期
    选择「周报」或「月报」，并选择对应的日期范围（弹窗宽度 700px，日期选择器宽度 340px）
  3. 关联工作记录
    点击「加载工作记录」拉取周期内已提交的工作记录，通过 checkbox 勾选需要关联的记录，支持全选和展开更多
  4. 编写内容
    直接手动输入正文，或点击「AI 生成」让 AI 自动生成
  5. 提交或保存
    保存草稿 暂存后续编辑
    提交 提交给上级审批（自动关联周期内已提交的工作记录到 report_record 表）

管理者的「含下属」功能

管理者和超级管理员在创建/编辑汇报时，可以勾选「含下属（周期全覆盖）」选项。勾选后，系统会加载直接下属在周期范围内的已通过汇报内容，作为编写参考。跨级下属不会加载（因为中间级已汇总了其下属内容）。下属汇报的周期必须完全落在当前筛选周期范围内才会被加载。

汇报状态与操作

状态（后端值）可执行操作说明
草稿（0） 编辑、提交、删除 尚未提交，可自由编辑
已提交（1） 查看审批日志 已提交给上级，等待审批
已退回（2） 编辑后重新提交 上级退回，修改后可再次提交
已通过（3） 查看详情、查看审批日志 上级已审核通过，汇报完成

编辑草稿注意事项

场景说明
变更类型/周期如果修改报告类型（周报/月报）或周期范围，系统自动重载关联的工作记录
AI 生成中编辑已有草稿时，AI 生成期间内容输入框被禁用，生成完毕后恢复。新建弹窗中 AI 生成时不锁定输入框
保存记录关联创建时选中的工作记录保存后自动关联，类型或周期变更后保存也会更新关联

顶部筛选

支持「全部 / 草稿 / 进行中 / 已通过」四个 radio 按钮快速切换视图。

审批日志

每份汇报的审批操作（退回原因、备注等）以时间线形式记录，可随时查看。

🤖 AI 生成汇报

AI 生成功能基于 OpenAI 兼容 API，将周期内的工作记录自动组织为汇报内容，大幅提升编写效率。

使用条件

需要超级管理员先在「AI 配置」页面设置好 AI 接口（接口地址、模型、密钥），所有员工方可使用。

使用步骤

  1. 新建汇报或编辑草稿
    进入汇报编辑界面
  2. 点击「AI 生成」按钮
    系统读取周期内已提交的工作记录作为输入，调用 AI 生成汇报内容。生成过程中按钮显示「AI 生成中...」并有弹窗提示
  3. 审阅并提交
    AI 生成的内容自动填入正文输入框，人工审阅修改后提交
⚠️ AI 生成的内容仅供参考，请务必人工审阅后再提交。如果「AI 生成」按钮灰色不可点击，说明 AI 接口尚未配置，请联系超级管理员处理。

✅ 审批管理

👥 本功能对管理者和超级管理员开放。普通员工看不到此菜单。

审批管理是上级审核下属周期汇报的核心功能。

审批范围

选项说明
全部下属查看所有有汇报关系的下属汇报（含间接下属，需跨级权限）
直属下属仅查看直接汇报给自己的下属的汇报

状态筛选

选项说明
待审核（1）下属已提交、等待审批的汇报（默认视图）
已退回（2）你曾经退回过的汇报
已通过（3）已审核通过的汇报
全部所有下属的各类汇报

审批操作

操作说明
通过 审核通过，汇报状态变为「已通过」。可选填审批意见，员工会收到通过通知
退回 填写退回原因（必填），汇报状态变为「已退回」。员工可查看原因并修改后重新提交
备注 添加审批备注，不改变汇报状态。可用于补充建议

权限规则

  • 操作按钮仅对当前汇报单的接收人（receiver_id）显示
  • 超级管理员可以审批任何人提交的汇报
  • 管理者只能审批提交给自己的汇报

审批日志

每份汇报的所有审批操作（退回原因、通过意见、备注）以时间线形式记录，操作人、操作类型、时间、内容均可追溯。

功能说明

⚙️ AI 配置

🔧 本功能仅超级管理员可访问。普通员工和管理者均看不到此页面。

配置 AI 生成汇报所需的接口参数。支持所有 OpenAI 兼容 API（含国内模型如智谱 GLM、DeepSeek、通义千问、Kimi 等）。

配置项

配置项说明示例
接口地址AI 服务的 API 地址https://api.openai.com/v1/chat/completions
模型名称使用的模型标识gpt-4o-mini
API 密钥接口认证密钥（只显示前4位和后4位）sk-xxxx...xxxx
系统提示词控制 AI 生成汇报的格式和风格你是一个工作汇报助手...

操作说明

  1. 填写配置
    填入接口地址、模型名称、API 密钥和系统提示词
  2. 测试连接
    点击「测试连接」验证 API 是否能正常响应，会显示成功或失败提示
  3. 保存
    测试通过后点击「保存」即可生效

修改密钥

如需修改完整密钥，点击「修改密钥」按钮，在弹窗中输入完整的新密钥。

⚠️ 接口地址使用 localhost 可能解析到 IPv6 导致连接失败，建议使用 127.0.0.1。

❓ 常见问题

Q1: 员工提交了记录，但上级看不到？

请检查是否已为该员工配置汇报关系。路径：超级管理员 → 汇报关系管理 → 添加关系。

Q2: 如何修改密码？

点击右上角头像 → 修改密码。输入旧密码和新密码即可。遗忘请联系超级管理员重置。

Q3: AI 生成按钮灰色不可点击？

需超级管理员先在「AI 配置」页面完成 API 地址、模型名称、密钥的配置并保存。

Q4: 草稿数据会丢失吗？

草稿存储于服务器数据库，不会因浏览器缓存清除或电脑重启而丢失。

Q5: 忘记密码怎么办？

请联系超级管理员重置密码。管理员可在用户管理中编辑用户信息并设置新密码。

Q6: 周报周期是如何划分的？

按自然周划分（周一至周日），提交工作记录时自动归入记录日期所在周的周报草稿。

Q7: 被退回的汇报如何处理？

在周期汇报页面找到被退回的汇报，点击编辑修改内容后点击「重新提交」即可再次提交给上级。

Q8: 管理者看不到某些下属的汇报？

请确认：1）该下属的汇报已提交（不是草稿）；2）汇报关系已正确配置（你是该下属的接收人）。如配置正确仍看不到，请联系超级管理员排查。

Q9: 如何导出工作记录？

在工作记录页面筛选好时间段后，点击「导出 Excel」或「导出 JSON」，即可将当前筛选结果保存到本地。

Q10: 新建用户时可以选择哪些角色？

只能创建「普通员工」和「管理者」。超级管理员账号需要由系统预置，不能通过界面创建。
