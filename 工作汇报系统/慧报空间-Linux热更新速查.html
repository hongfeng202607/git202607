🔥 慧报空间 · Linux 热更新速查

生产环境部署参考，适用路径：/data/Work Report System/
📌 基本原则
改了什么 更新方式 是否需要构建
backend/routes/*.js 直接替换文件 → 重启服务 ❌ 不需要
frontend/src/**/*.vue 必须重新构建，服务器只读 dist/ ✅ 必须 build
frontend/src/**/*.js 必须重新构建 ✅ 必须 build
backend/*.json（package.json） 替换后重新 npm install ⚠️ 需重装依赖
🔧 完整操作流程
服务器信息：10.61.190.27 · 用户 appadmin · systemd 服务名 huibao
  1. 1
    停止服务
    sudo systemctl stop huibao
  2. 2
    替换后端文件（如 user.js）
    # 在本地操作（Git Bash）
scp backend/routes/user.js appadmin@10.61.190.27:"/data/Work Report System/backend/routes/"
  3. 3
    如果改了前端 → 重新构建
    # 在服务器上执行
cd "/data/Work Report System/frontend"
npm run build
    💡 改过后端可以跳过此步
  4. 4
    启动服务
    sudo systemctl start huibao
  5. 5
    确认状态 + 查看日志
    sudo systemctl status huibao
journalctl -u huibao -n 20 --no-pager
日志中看到 Frontend dist loaded 和 started on port 8901 即为成功
⚠️ 常见踩坑
只传 .vue 不构建
生产环境读 dist/ 不读 .vue，必须 npm run build
后端改 .js 不必构建
Node.js 直接跑 .js 源文件，替换后重启即生效
用 nohup 关 SSH 就掉
只能用 systemctl 管理，不要用 nohup node app.js &
没改 package.json 不需 npm install
node_modules 是跨平台不兼容的，只在首次部署时装一次
⚡ 常用命令速查
操作命令
重启服务sudo systemctl restart huibao
停止服务sudo systemctl stop huibao
启动服务sudo systemctl start huibao
查看状态sudo systemctl status huibao
查看实时日志journalctl -u huibao -f
查看最近 50 行日志journalctl -u huibao -n 50 --no-pager
