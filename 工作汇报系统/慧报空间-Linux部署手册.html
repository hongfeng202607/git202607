安装 PostgreSQL 16

# 查看系统自带源中是否有 PostgreSQL
dnf list available | grep postgresql
# 安装 PostgreSQL 16 服务端和客户端
sudo dnf install -y postgresql-server postgresql-contrib
# 初始化数据库
sudo /usr/bin/postgresql-setup --initdb
# 启动 PostgreSQL 并设置开机自启
sudo systemctl start postgresql
sudo systemctl enable postgresql
# 验证状态
sudo systemctl status postgresql
✓ PostgreSQL 16 安装完成，正在运行

创建数据库并恢复数据

# 设置 postgres 用户密码
sudo -u postgres psql -c "ALTER USER postgres WITH PASSWORD '123456';"
# 创建数据库
sudo -u postgres createdb work_report_db
# 配置允许密码登录（修改认证方式）
sudo sed -i 's/ident/md5/g; s/peer/md5/g' /var/lib/pgsql/data/pg_hba.conf
sudo systemctl restart postgresql
# 恢复数据（从 Windows 导出的备份还原到 Linux）
export PGPASSWORD=123456
psql -h 127.0.0.1 -U postgres -d work_report_db -f "/data/Work Report System/backend/backup/慧报空间部署备份_2026-05-29T0335.sql"
✓ 数据库创建完成，数据已恢复
⚠ 备份文件在 Windows 上用 pg_dump 导出，传输到 Linux 后直接 psql -f 恢复即可

配置 systemd 永久运行（关键步骤）

6.1 创建 systemd 服务文件

sudo vi /etc/systemd/system/huibao.service

写入以下内容（直接复制）：

[Unit]
Description=慧报空间
After=network.target postgresql.service

[Service]
Type=simple
WorkingDirectory=/data/Work Report System/backend
ExecStart=/usr/bin/node /data/Work\ Report\ System/backend/app.js
Environment=PORT=8901
Environment=DB_HOST=127.0.0.1
Environment=DB_PORT=5432
Environment=DB_NAME=work_report_db
Environment=DB_USER=postgres
Environment=DB_PASSWORD=123456
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

保存退出：按 Esc → 输入 :wq → 回车

6.2 生效并启动服务（4 条核心命令）

# 1. 重载 systemd，识别新服务
sudo systemctl daemon-reload
# 2. 启动服务
sudo systemctl start huibao
# 3. 设置开机自启
sudo systemctl enable huibao
# 4. 查看状态
sudo systemctl status huibao
✓ 看到 active (running) 即为成功

6.3 验证 API

开放防火墙端口

# 开放 8901 端口
sudo firewall-cmd --add-port=8901/tcp --permanent
sudo firewall-cmd --reload
# 确认端口已开放
sudo firewall-cmd --list-ports
✓ 输出应包含 8901/tcp

常见问题

Q: 服务启动失败，如何排查？

journalctl -u huibao -n 50 --no-pager

Q: 外部浏览器访问不通？

排查顺序：

# 1. 服务是否在监听
ss -tlnp | grep 8901
# 2. 本机是否正常
curl -s http://127.0.0.1:8901 | head -5
# 3. 防火墙是否开放
sudo firewall-cmd --list-ports

如果以上都正常，联系 IT 检查云平台安全组/网络安全策略。

Q: 如何切换到其他端口？

修改 huibao.service 中的 PORT 环境变量，更新防火墙规则，重启服务即可。
