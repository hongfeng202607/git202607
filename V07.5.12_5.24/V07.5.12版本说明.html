📌 版本概述

V07.5.12 是继 V07.5.10（增AI配置版）之后的重要架构升级版本，代号「智能款·增AI配置·架构升级版」。本次升级在完整保留 V07.5.10 全部功能（AI录入、AI智能搜索、三级权限、AI可视化配置、星标、查重）的基础之上，完成了三项重大架构升级：数据库从 SQLite 迁移至 PostgreSQL、后端代码重构为 Flask 蓝图架构、权限体系升级为 RBAC（基于角色的权限控制），并新增了提出人字段、角色管理、数据权限、换肤系统、仪表盘改版、PWA支持等功能。

版本代号：智能款·增AI配置·架构升级版

发布日期：2026年5月17日

最后更新：2026年5月24日（新增换肤系统、侧边栏导航、设置面板改版、PWA支持、数据权限、仪表盘改版、操作日志可视化）

技术栈：Flask 3.x（蓝图架构）+ PostgreSQL 14+ + HTML5 + JavaScript + OpenAI 兼容 API

🔥 V07.5.12 核心升级内容

1. 🗄️ 数据库迁移至 PostgreSQL 架构升级

  • 底层数据库从 SQLite 升级至 PostgreSQL 14+（psycopg2 驱动，本地连接 localhost:5432，数据库：knowledge_base）
  • SQL 语法全面迁移：占位符 ? 改为 %s，自增主键由 AUTOINCREMENT 改为 SERIAL，唯一冲突处理由 INSERT OR IGNORE 改为 ON CONFLICT DO NOTHING
  • DBConnection 封装类统一管理连接、事务、游标（RealDictCursor），保持与旧 sqlite3 API 调用风格兼容
  • 自动建表：系统首次启动时通过 CREATE TABLE IF NOT EXISTS + CREATE INDEX IF NOT EXISTS 建立全部表结构及5个核心索引
  • 数据库备份由 shutil 拷贝 .db 文件升级至 pg_dump 导出 .sql 文件，自动保留最近10份，存入 Backup/ 目录
  • 旧字段兼容迁移：通过 information_schema.columns 检查列是否存在，不存在则 ALTER TABLE ADD COLUMN，避免已有数据库报错

2. 🏗️ Flask 蓝图架构重构 架构升级

  • 后端代码从单文件 app.py 重构为 blueprints/ 蓝图目录，按功能模块拆分
  • db.py：数据库层（DBConnection封装、RBAC初始化、备份、AI配置读取）
  • auth.py：认证与用户管理（登录/登出/密码修改/账号CRUD/权限装饰器）
  • knowledge.py：知识记录增删改查（搜索/新增/编辑/回收站/查重/星标/统计/仪表盘/日志）
  • ai.py：AI功能（AI配置管理/AI解析/语义搜索）
  • attachment.py：附件管理（上传/下载/绑定/删除）
  • category.py：分类管理（获取/保存分类）
  • role.py：角色管理（角色CRUD/权限绑定）
  • data_permission.py：数据权限（开关/授权/查询）

3. 🛡️ 权限体系升级：RBAC 新增

  • 新增三张权限表：roles（角色表）、permissions（权限表：14个权限码）、role_permissions（角色权限关联表）
  • 所有接口权限校验从 if g.user["role"] == "admin" 升级为 @require_permission("permission.code") 装饰器
  • 系统内置3个系统角色（不可删除）：admin（全权）/ editor（编辑员）/ user（普通用户）
  • 管理员可新增自定义角色，动态勾选权限码，绑定到用户；删除角色前检查是否有用户仍在使用
  • 登录返回新增 permissions 权限列表，前端可精确控制按钮可见性
  • super_admin（isSadmin）前端豁免所有权限检查；后端 require_permission 也加 _is_sadmin 豁免
  • 权限依赖联动：勾选/取消自动联动（edit→view, delete→view, permanent_del→recycle.view 等）
  • 旧用户自动迁移：启动时扫描 role_id IS NULL 的旧用户，根据 role 字段匹配系统角色并补全 role_id

4. 👤 提出人（proposer）字段分离 新增

  • 知识记录表新增 proposer 字段（提出人/报障人/申请人），与 submitter（系统自动记录的提交人账号）明确区分
  • 新增知识时，前端表单新增「提出人」输入框，手动填写或 AI 自动识别提取
  • 搜索中 proposer 也作为匹配字段之一，支持按提出人关键词检索
  • AI智能录入：系统提示词新增提出人提取规则，从文本中识别报修/申请人等，AI返回字段 creator 映射为 proposer

5. 🎯 搜索加权评分升级 优化

  • 搜索结果排序从「ID倒序」升级为 加权相关度评分：标题相似度 × 0.6 + 内容相似度 × 0.4（使用 difflib.SequenceMatcher）
  • 语义搜索（AI助手）中，召回后同样应用加权评分排序，优先展示最相关的记录
  • 分词规则优化：仅对超过4字的「纯中文」片段做滑动窗口拆分，含英文字母的词保持整体匹配，减少误召回
  • 搜索范围扩展：proposer 字段也加入搜索条件（匹配 question/solution/remark/submitter/proposer 共5个字段）

6. 🔑 角色管理接口 新增

  • 新增 GET /api/roles：获取全部角色及其权限列表（需 user.manage 权限）
  • 新增 POST /api/roles/add：新增自定义角色，传入名称+描述+权限码数组
  • 新增 POST /api/roles/update：更新角色名称/描述/权限绑定（全量替换）
  • 新增 POST /api/roles/del：删除自定义角色（系统内置角色不可删，有用户绑定时拒绝删除）
  • 新增 GET /api/permissions：获取全部权限定义（按分组返回）
  • 新增 POST /user/role：修改指定用户的角色绑定，修改后踢下线强制重新登录

7. 🤖 AI 智能录入 JSON 截断自动修复 修复

  • 当 AI 返回的 JSON 因 max_tokens 限制被截断时，自动尝试三级修复：①补全末尾引号和花括号 ②正则提取完整 JSON ③抛出异常提示用户重试

8. 📋 数据管理集成到主页 新增

📝 V07.5.10 升级至 V07.5.12 注意事项

  • 1. 环境准备（必须）：安装 PostgreSQL 14+，创建数据库 knowledge_base，用户名 postgres，密码 123456（或通过环境变量 KB_DB_USER / KB_DB_PASSWORD 配置）
  • 2. 安装 Python 依赖：pip install psycopg2-binary pandas openpyxl flask requests
  • 3. 旧数据迁移（可选）：如需从 SQLite 迁移历史数据，使用数据库工具 DBManager.exe 先导出 Excel，启动新版后通过 DBManager 导入
  • 4. 目录结构变化：app.py 仅作为入口，实际逻辑全部在 blueprints/ 目录下，部署时需确保 blueprints/ 目录完整
  • 5. pg_dump 路径：默认 D:\PostgreSQL\bin\pg_dump.exe，可通过环境变量 KB_PG_BIN 覆盖
  • 6. 首次启动：程序自动建表、初始化RBAC角色权限、创建admin账号（已有则跳过），旧用户自动迁移到新角色体系
  • 7. AI配置：V07.5.10 的 sys_config AI配置数据会随数据库迁移一起带过来；否则首次启动后需重新在前端配置
