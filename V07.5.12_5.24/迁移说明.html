迁移概览 — 五步流程

1
目标机器环境准备

安装 Python 3.10+、PostgreSQL 15+，pip 安装依赖包

2
数据库迁移

在源机器导出 .sql 备份，在目标机器新建库并恢复

3
代码配置修改

修改 PG_BIN 路径（或设置环境变量），确认数据库连接参数

4
文件与附件迁移

整体复制项目文件夹，并复制 uploads/ 目录中的附件文件

5
启动与验证

运行 python launcher.py，登录系统验证功能

⚠️
迁移前务必在源机器先做一次完整数据库备份（Backup/ 文件夹中的 .sql 文件），确保有可用的还原点。

第一步：目标机器环境准备

1.1 Python 环境

组件要求说明
Python3.10 或以上（推荐 3.12）官网下载安装，勾选"Add Python to PATH"
pip随 Python 自动安装国内可配置清华镜像加速

安装依赖包（在项目目录下执行）：

pip install -r requirements.txt
# 如果网速慢，使用国内镜像：
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

依赖清单（requirements.txt）：

包名版本要求用途
flask>= 3.0Web 框架
psycopg2-binary>= 2.9PostgreSQL 驱动
pandas>= 2.0Excel 导入导出
openpyxl>= 3.1Excel 文件读写
requests>= 2.31AI 接口调用

1.2 PostgreSQL 安装

组件要求说明
PostgreSQL15 或以上（推荐 16/17）安装时设置用户 postgres，密码建议统一
安装路径记住 bin 目录路径默认通常在 C:\Program Files\PostgreSQL\17\bin
💡
安装完成后验证是否可用：
打开 CMD，执行 psql --version。
若提示"找不到命令"，需要将 bin 目录手动加入系统 PATH，或在后续步骤中通过 KB_PG_BIN 环境变量指定路径。

1.3 创建数据库

# 在目标机器上，用 psql 创建数据库（以下命令在 CMD 中执行）
psql -U postgres
-- 在 psql 提示符下执行：
CREATE DATABASE knowledge_base;
\q

或者直接一行命令：

createdb -U postgres knowledge_base

第二步：数据库迁移（PostgreSQL）

2.1 在源机器导出数据

必须 迁移前在源机器做备份，有两种方式：

方式一：使用系统内置工具（推荐，最简单）

打开浏览器访问知识库系统，进入「数据管理」→「备份管理」→ 点击「立即备份」。备份文件自动生成在项目的 Backup/ 文件夹中。

方式二：双击 手动备份.py 脚本

备份文件同样保存在 Backup/ 文件夹。

方式三：命令行手动导出

D:\PostgreSQL\bin\pg_dump.exe -U postgres -h localhost -d knowledge_base -f E:\备份.sql --no-owner

2.2 将备份文件复制到目标机器

将导出的 .sql 文件（位于 Backup/ 目录）复制到目标机器项目文件夹的 Backup/ 目录中。

2.3 在目标机器恢复数据

确保已完成第一步（创建了 knowledge_base 库），然后：

方式一：使用系统内置恢复功能（推荐）

启动服务后，进入「数据管理」→「备份管理」→ 选择备份文件 → 点击「恢复」。

方式二：双击 恢复备份.py 脚本

按提示选择备份文件编号，自动执行恢复。

方式三：命令行手动恢复

# Step 1: 清空现有数据
C:\Program Files\PostgreSQL\17\bin\psql.exe -U postgres -d knowledge_base -c "DROP SCHEMA public CASCADE; CREATE SCHEMA public;"

# Step 2: 导入备份文件
C:\Program Files\PostgreSQL\17\bin\psql.exe -U postgres -d knowledge_base -f D:\知识库\Backup\手动备份_20260523.sql
✅
恢复成功后系统会自动重新初始化 RBAC 角色权限和管理员账号，无需手动执行建表脚本。
🚫
如果目标机器是全新空库（未做过任何初始化），建议先启动系统让其自动建表，然后再通过"恢复"功能覆盖数据，避免外键冲突。

第三步：代码配置修改 必改

🚫
以下涉及 PostgreSQL 工具路径的代码是写死的默认值，迁移到不同机器后必须修改，否则备份/恢复功能报 [WinError 2] 系统找不到指定的文件。

3.1 需要修改的文件及位置

文件行号原值（示例）需改为
app.py 第 9 行 r'D:\PostgreSQL\bin' 改为目标机器实际 bin 路径
blueprints/db.py 第 35 行 r'D:\PostgreSQL\bin' 同上，保持一致
手动备份.py 第 15 行 r'D:\PostgreSQL\bin' 同上
恢复备份.py 第 14 行 r'D:\PostgreSQL\bin' 同上

3.2 修改方法

方法A（最推荐）：设置系统环境变量，不用改代码

# CMD 临时设置（仅当前窗口有效）
set KB_PG_BIN=C:\Program Files\PostgreSQL\17\bin
python app.py

# PowerShell 临时设置
$env:KB_PG_BIN = "C:\Program Files\PostgreSQL\17\bin"
python app.py

也可以在 Windows 系统设置 → 环境变量 → 新建系统变量 KB_PG_BIN，设置为 PostgreSQL bin 目录路径，永久生效，无需每次手动设置。

方法B：直接修改代码中的默认值

用记事本或 VSCode 打开上表中的文件，将 r'D:\PostgreSQL\bin' 改为目标机器实际路径，例如：

# 原写法：
PG_BIN = os.environ.get('KB_PG_BIN', r'D:\PostgreSQL\bin')

# 改为（示例，替换为你实际的路径）：
PG_BIN = os.environ.get('KB_PG_BIN', r'C:\Program Files\PostgreSQL\17\bin')
⚠️
Windows 常见 PostgreSQL bin 路径参考：
C:\Program Files\PostgreSQL\17\bin（17版，默认安装位置）
C:\Program Files\PostgreSQL\16\bin（16版）
D:\PostgreSQL\bin（自定义安装到 D 盘）
不确定时可在 CMD 执行 where psql 查看实际路径。

3.3 数据库连接参数（按需修改）

数据库连接参数通过环境变量控制，代码默认值如下，如目标机器一致则无需修改：

环境变量默认值说明
KB_DB_HOSTlocalhost数据库主机，本机部署无需改
KB_DB_PORT5432PostgreSQL 默认端口
KB_DB_USERpostgres数据库用户名
KB_DB_PASSWORD123456建议修改 目标机器密码若不同必须设置
KB_DB_NAMEknowledge_base数据库名，通常不用改
KB_PG_BIND:\PostgreSQL\bin必须确认 PostgreSQL bin 目录

如果目标机器 PostgreSQL 密码不同，设置环境变量：

set KB_DB_PASSWORD=你的新密码

3.4 端口确认

服务默认运行在 8901 端口（app.py 第 504 行）。如需修改端口，直接修改该行：

app.run(host="0.0.0.0", port=8901, debug=False, use_reloader=False)

附录A：推荐用环境变量管理配置

所有配置支持通过环境变量覆盖，强烈建议生产环境用环境变量而不是修改代码，这样迁移时无需改代码：

Windows 系统环境变量（永久生效）

控制面板 → 系统 → 高级系统设置 → 环境变量 → 系统变量 → 新建：

变量名值（示例）
KB_PG_BINC:\Program Files\PostgreSQL\17\bin
KB_DB_PASSWORD你的数据库密码
KB_DB_HOSTlocalhost（远程数据库时填IP）
KB_DB_PORT5432
KB_DB_USERpostgres
KB_DB_NAMEknowledge_base

批处理启动脚本（推荐新建）

可以在项目目录新建一个 start.bat 文件，集中管理配置：

@echo off
chcp 65001
set KB_PG_BIN=C:\Program Files\PostgreSQL\17\bin
set KB_DB_PASSWORD=你的密码
set KB_DB_HOST=localhost
python launcher.py

双击 start.bat 即可启动，无需记路径，也无需修改代码。

附录B：常见问题 FAQ

Q1：启动报 [WinError 2] 系统找不到指定的文件

原因： KB_PG_BIN 路径配置不正确，找不到 psql.exe 或 pg_dump.exe。

解决： 在 CMD 执行 where psql 找到实际路径，然后设置 KB_PG_BIN 环境变量或修改代码中的默认路径。

Q2：启动报 psycopg2.OperationalError: connection refused

原因： PostgreSQL 服务未启动，或连接参数（密码/端口）不对。

解决： 在服务管理器（services.msc）中确认 PostgreSQL 服务已启动；检查密码是否正确（设置 KB_DB_PASSWORD）。

Q3：登录后知识记录为空

原因： 数据库恢复步骤未执行，或恢复的是空备份。

解决： 确认 Backup/ 目录中有 .sql 备份文件，按第二步操作恢复数据。

Q4：附件图片/文件打不开

原因： uploads/ 目录中的文件没有一起迁移。

解决： 将源机器 uploads/ 目录的全部文件复制到目标机器对应目录。

Q5：pip 安装 psycopg2-binary 失败

原因： 网络问题，或 Python 版本不兼容。

解决： 换国内镜像：pip install psycopg2-binary -i https://pypi.tuna.tsinghua.edu.cn/simple；确认 Python 版本 >= 3.10。

Q6：页面打开后没有样式（CSS/JS 加载失败）

原因： static/ 目录未迁移，或迁移不完整。

解决： 确认 static/style.css 和 static/app.js 两个文件都存在。

Q7：源机器备份导入目标机器后，管理员密码/角色失效

原因： 正常现象，系统恢复后会重新初始化 RBAC 和管理员账号。

解决： 用 admin / 123456 登录即可。如之前修改过密码，可在系统内重新设置，或在数据库中手动更新。

Q8：运行在 Linux/Mac 上会有什么问题？

注意： 项目 PostgreSQL 工具路径默认带 .exe 后缀（Windows 格式），在 Linux/Mac 下需要去掉 .exe 后缀，并将 pg_dump.exe 改为 pg_dump、psql.exe 改为 psql。
