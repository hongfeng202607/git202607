## 2026-06-06 工作记录（追加）

### 系统无法启动排查与修复
**问题1：launcher.py 的 emoji 编码崩溃**
- 根因：Windows 控制台默认 GBK 编码，`🌐`(U+1F310) 和 `✅`(U+2705) 无法编码，print 直接抛出 UnicodeEncodeError，Flask 未能启动
- 修复：在 launcher.py 开头添加 `sys.stdout.reconfigure(encoding='utf-8', errors='replace')`

**问题2：PostgreSQL 数据库服务未运行**
- 根因：postgresql-x64-18 服务处于 Stopped 状态
- 修复中发现的二级问题：data 目录缺失多个标准子目录（pg_notify、pg_stat、pg_stat_tmp、pg_dynshmem、pg_replslot、pg_serial、pg_commit_ts、pg_tblspc、pg_twophase、pg_snapshots、pg_logical/mappings）——猜测是上次 PG 异常关闭导致目录丢失
- 解决：用 Python 逐一创建缺失目录 → pg_ctl start 成功
- 额外：将 PG 服务设为 Automatic 自动启动，避免每次手动启
- 更新具体方案中 DB_CONFIG 代码示例，加上环境变量读取说明
- 清理 db_config.ini：移除过时 SQLite 配置和注释，新增 KB_PG_BIN 环境变量文档
